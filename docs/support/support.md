# Support

**Last Updated**: January 2, 2026

## Support Scope

Support for Firsttry is provided on a **best-effort basis**. This document outlines what support is available and what is excluded.

## Contact

For support inquiries:
- **Email**: contact@firsttry.run

## What Support Covers

Support may assist with:
- Questions about App functionality and features
- Clarification of App behavior and output
- Guidance on configuration options
- Bug reports and issue triage
- Documentation clarification

## What Support Does NOT Cover

Support explicitly does NOT include:
- **Compliance Interpretation**: The App provides informational data only. Support cannot provide legal, regulatory, or compliance advice
- **Jira Administration**: Questions about Jira itself should be directed to Atlassian Support
- **Custom Development**: Feature requests or custom modifications are evaluated on a case-by-case basis but are not guaranteed
- **24/7 Availability**: Support is not available around the clock
- **Guaranteed Response Times**: No SLA or response time guarantees are provided
- **Implementation Services**: Support does not include hands-on configuration or deployment assistance
- **Third-Party Integration**: Support for integrating with tools outside the Atlassian ecosystem is not provided

## Support Process

1. **Submit Request**: Email contact@firsttry.run with:
   - Clear description of the issue
   - Steps to reproduce (if applicable)
   - Expected vs actual behavior
   - Jira Cloud environment details (if relevant)

2. **Triage**: Requests are reviewed and prioritized
   - Critical bugs affecting core functionality receive priority
   - Feature requests are evaluated separately

3. **Response**: You will receive a response on a best-effort timeline
   - No guaranteed response times
   - Complex issues may require additional investigation

4. **Resolution**: 
   - Bugs may be fixed in future releases
   - Feature requests may be considered for roadmap inclusion
   - Workarounds may be suggested where applicable

## Known Limitations

The App has known limitations:
- Depends on Atlassian Forge platform availability
- Storage is subject to Forge platform quotas
- Performance depends on Jira instance size and complexity
- Some advanced features may require specific Jira Cloud plans

See documentation for details on limitations and workarounds.

## Bug Reporting

When reporting bugs, include:
- Detailed steps to reproduce
- Expected behavior vs actual behavior
- Any error messages or logs (if visible)
- Jira Cloud environment context

**Note**: Do not include sensitive data, credentials, or personal information in bug reports.

## Feature Requests

Feature requests can be submitted via email. Requests are:
- Reviewed for feasibility and alignment with App goals
- Prioritized based on user impact and technical complexity
- Not guaranteed for implementation

## Disclaimer

Support is provided on a best-effort, as-available basis without guarantees. The App is provided "as is" without warranty of any kind. See Terms of Service for full disclaimers.

---

**Important**: For issues with the Atlassian Forge platform itself, contact Atlassian Support directly. This support channel is for App-specific issues only.
